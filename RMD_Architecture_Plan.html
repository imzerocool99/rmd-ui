1. Executive Summary

The Intelligent RMD Liquidation Agent is an AI-powered platform that automates IRA Required Minimum Distribution (RMD) processing for wealth management firms. It combines IRS regulatory logic, tax-efficient portfolio analysis, autonomous trade execution, and a natural language AI assistant to transform a complex manual process into a fully automated, explainable, and client-centric workflow.

Two Execution Modes:
  • Option 1 — Cash Distribution: Sells selected assets, transfers cash to the client's bank account.
  • Option 2 — In-Kind Transfer: Moves shares from IRA to taxable brokerage. Client retains market exposure while satisfying the RMD requirement.

The solution is built entirely with free, open-source tools and runs 100% locally on a developer's PC — no cloud subscription required.

2. Business Problem

Traditional IRA RMD processing is manual, slow, and carries significant compliance and financial risk:

  • The IRS mandates annual RMDs after age 73. Failure triggers a 25% excise tax on the undistributed amount.
  • Poor asset selection generates unnecessary capital gains and tax drag, reducing after-tax returns for clients.
  • Operations teams spend hours per account on manual calculations, trade tickets, and compliance documentation.
  • Clients lack transparency — they don't know why specific assets were chosen for distribution.
  • The process does not scale: as the number of aging IRA accounts grows, operational burden grows proportionally.

3. Solution Overview

The agent automatically executes the full RMD workflow end-to-end:

  1. Determines RMD eligibility and calculates the required distribution amount using the IRS formula.
  2. Analyzes the portfolio — scores each asset for tax efficiency, performance, and concentration risk.
  3. Uses an AI model (Ollama/Phi-3, running locally) to choose the optimal distribution strategy.
  4. Executes sell orders or in-kind transfers via the Alpaca paper trading API.
  5. Generates a plain-English explanation of every decision for the client and advisor.
  6. Logs a full audit trail for compliance and regulatory review.

5. Core Components

5.1 RMD Calculation Engine (RMDService.java)

Calculates the Required Minimum Distribution using the IRS Uniform Lifetime Table formula:

RMD = Prior Year-End IRA Balance ÷ IRS Life Expectancy Factor
Example: $500,000 balance, age 75, factor = 24.6 → RMD = $20,325.20

Validates account type (Traditional IRA, Inherited IRA), age eligibility, and IRS rule compliance.

5.2 Asset Ranking & Selection (AssetRankingService, AssetSelectionService)

Scores each portfolio holding for tax efficiency. Scoring preference order:

  • Assets with unrealized losses (best — generates tax loss harvesting benefit).
  • Underperforming assets (second — reduces portfolio drag).
  • Over-concentrated positions (third — natural rebalancing opportunity).
  • High-gain, long-term holds (last — avoid disturbing best performers).

Allocates exact share quantities to match the RMD amount, minimizing remainder.

5.3 Tax Optimization Engine (TaxService)

Evaluates tax impact of each candidate asset: capital gains vs losses, cost basis, ordinary income treatment of IRA withdrawals, wash-sale risk, and portfolio rebalancing impact. Outputs a ranked liquidation strategy with estimated net tax impact.

5.4 Trade Execution via Alpaca (AlpacaMCP, AlpacaClient)

Executes all trades via the Alpaca paper trading REST API. Supports market orders, position updates, NAV recalculation, and trade confirmation. In-kind transfers are simulated (Alpaca does not support IRA-to-brokerage transfers in paper mode).

5.5 AI Layer — Ollama + Phi-3 (AzureLLM.java → now Ollama)

The LLM layer has two capabilities:

  • Strategy Decision: Chooses between cash sale and in-kind transfer based on market prediction context. Falls back to rule-based logic if Ollama is offline.
  • AI Chat Assistant: Answers client questions in plain English using the actual agent context (RMD amount, selected assets, strategy). Runs locally via Ollama on port 11434.

5.6 Memory & Audit (MemoryService, MonitorService)

MemoryService stores per-client history of past RMD decisions and execution results. MonitorService logs every agent action with timestamps and exposes them via the /agent/logs REST endpoint.

5.7 Frontend Dashboard (rmd-ui)

A single-page HTML5/JS application with five sections: Dashboard (KPI cards + portfolio chart), Portfolio (holdings table + performance chart), Agent Results (selected assets + RMD coverage chart), AI Assistant (chat interface with quick-question buttons), and Logs (real-time agent log viewer). Served on port 4500.

11. Local Deployment Guide

Step 1 — Install Java 17

winget install Microsoft.OpenJDK.17 --accept-package-agreements --accept-source-agreements

Step 2 — Install Ollama (Local AI Engine)

Ollama runs AI models entirely on your local PC — no API key, no cloud, no cost.

# Install via winget winget install Ollama.Ollama --accept-package-agreements --accept-source-agreements # Pull the Phi-3 model (3.8B — fast on CPU, no GPU required) ollama pull phi3 # Verify ollama list

Ollama runs as a Windows background service on port 11434. Verify at http://localhost:11434.

Alternative free models:

  • ollama pull phi3 — Microsoft Phi-3, 3.8B (recommended for this POC)
  • ollama pull llama3.2 — Meta LLaMA 3.2, 3B, excellent reasoning
  • ollama pull mistral — Mistral 7B, strong for structured tasks

Step 3 — Install Maven & Node.js

# Maven: download and extract Invoke-WebRequest -Uri "https://archive.apache.org/dist/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.zip" -OutFile "$env:USERPROFILE\maven.zip" -UseBasicParsing Expand-Archive "$env:USERPROFILE\maven.zip" -DestinationPath "$env:USERPROFILE\maven" $env:PATH = "$env:USERPROFILE\maven\apache-maven-3.9.6\bin;$env:PATH" # Node.js winget install OpenJS.NodeJS --accept-package-agreements --accept-source-agreements

Step 4 — Clone, Checkout Branch, Build

git clone https://github.com/imzerocool99/rmd-agent git clone https://github.com/imzerocool99/rmd-ui cd rmd-agent; git checkout feature/local-poc-ui cd ..\rmd-ui; git checkout feature/local-poc-ui # Build backend JAR cd ..\rmd-agent $env:JAVA_HOME = "C:\Program Files\Microsoft\jdk-17.0.19.10-hotspot" $env:PATH = "$env:JAVA_HOME\bin;$env:PATH" mvn clean package -DskipTests

Step 5 — Run All Three Services

12. KPIs & Success Metrics

  • % of RMDs completed automatically without human intervention.
  • Reduction in estimated tax impact vs naive (random) asset selection.
  • Client adoption rate — % who interact with the AI assistant.
  • Zero IRS penalty incidents across all managed accounts.
  • Processing time saved vs manual RMD workflow per account.
  • AI explanation satisfaction score (advisor and client feedback surveys).
