1. Executive Summary

The Intelligent RMD Liquidation Agent is an AI-powered platform that automates IRA Required Minimum Distribution (RMD) processing. It combines IRS regulatory logic, tax-efficient portfolio analysis, autonomous trade execution, and a context-aware AI chat assistant to transform a complex manual process into a fully automated, explainable, and client-centric workflow.

Two Execution Modes:
  • Cash Distribution: Sells selected assets, transfers cash to the client's bank account. Agent then suggests reinvestment products for that cash.
  • In-Kind Transfer: Moves shares from IRA to taxable brokerage. Client retains market exposure while satisfying the RMD.
Agent Response Time: ~4 seconds — rule-based logic is instant; Ollama/phi3 generates reinvestment advice asynchronously (4s budget, falls back to rule-based if exceeded).

2. Business Problem

  • IRS mandates annual RMDs after age 73 — failure triggers a 25% excise tax on the undistributed amount.
  • Poor asset selection generates unnecessary capital gains and tax drag.
  • Operations teams spend hours per account on manual calculations, trade tickets, and compliance documentation.
  • Clients lack transparency — they don't know why specific assets were chosen.
  • RMD cash sitting in a checking account earns nothing — no reinvestment guidance provided.
  • Process does not scale without proportional staffing increases.

3. Solution Overview

  1. Determines RMD eligibility and calculates required amount using IRS Uniform Lifetime Table.
  2. Analyzes portfolio across 10 asset classes — scores each asset for tax efficiency and performance.
  3. Selects assets with losses/underperformance first to minimize tax impact.
  4. Shows exact tax savings vs naive (random) asset selection — side-by-side comparison.
  5. Executes trades via Alpaca paper trading platform (simulated for POC).
  6. Suggests 7 reinvestment products for the distributed cash (HYSA, CD, T-Bills, Muni Bonds, Dividend ETF, Fixed Annuity, Money Market).
  7. Context-aware AI chat bot answers client questions based on whichever tab they are viewing.
  8. Full audit trail and agent execution logs for compliance.

5. Core Components

5.1 RMD Calculation Engine — RMDService.java

RMD = Prior Year-End IRA Balance ÷ IRS Life Expectancy Factor
Age 72 → factor 27.4  |  Age 75 → factor 24.6  |  Age 80 → factor 20.2
Example: $500,000 balance, age 75 → RMD = $20,325.20

5.2 Asset Ranking & Selection — AssetRankingService, AssetSelectionService

Scores all holdings for tax efficiency. Selection order (best to worst for liquidation):

  1. Assets with unrealized losses (tax loss harvesting benefit)
  2. Underperforming assets (reduces portfolio drag)
  3. Over-concentrated positions (natural rebalancing)
  4. High-gain, long-term holds (liquidated last — protect best performers)

Allocates exact share quantities to match the RMD amount with minimal remainder.

5.3 Tax Optimization Engine — TaxService.java

Evaluates unrealized gain/loss, cost basis, ordinary income treatment, wash-sale risk, and concentration risk per asset. Also runs a naive comparison (what would happen if the highest-gain assets were selected instead) to quantify the tax savings generated by the agent.

5.4 Reinvestment Engine — ReinvestmentSuggestionService.java NEW

After RMD cash hits the bank account, the agent generates 7 reinvestment product suggestions. Allocation percentages are adjusted based on client age and risk profile.

5.5 AI Layer — AzureLLM.java (Ollama/Phi-3)

Three levels of AI usage, each with a smart fallback:

FeatureUses Ollama?TimeoutWhy
Strategy decision (cash vs in-kind)❌ Rule-basedInstantSimple decision — market prediction context is enough
Reinvestment advice✅ Ollama async4 secondsRich personalized advice across 7 products — worth the wait
Chat assistant (/agent/chat)✅ Ollama4 secondsUser explicitly asking — expects a thoughtful response

Both Ollama calls use a 4-second timeout with smart rule-based fallback. Reinvestment advice fires in a CompletableFuture async thread so it runs in parallel with tax analysis — reducing total wait time.

5.6 Trade Execution — AlpacaMCP.java

POC mode: simulates trade execution locally — no external HTTP call, instant response. Production upgrade path: remove simulation flag → real Alpaca paper/live orders fire.

5.7 Memory & Audit — MemoryService, MonitorService

Stores per-client decision history. Every agent action is logged with timestamps, exposed via GET /agent/logs.

9. Agent Execution Flow — SENSE → THINK → PLAN → ACT → REPORT

1
SENSEReads client age, IRA balance, 13-asset portfolio across 10 asset classes, market prediction, and client preference (cash/in-kind) from request.
2
THINKCalculates RMD using IRS Uniform Lifetime Table. Scores every asset for tax efficiency — losses preferred first, then underperformers, then concentrated positions.
3
PLANAsset selector allocates exact quantities to match RMD amount. Rule-based logic decides cash vs in-kind. Tax analysis runs naive vs optimized comparison to quantify savings. Reinvestment engine generates 7 product suggestions.
4
ACTSimulates trade execution via AlpacaMCP (POC mode — instant, no HTTP call). In production: real Alpaca paper/live orders. Updates positions and NAV.
5
REPORTReturns full response: selected assets, tax savings, reinvestment suggestions, AI explanation. Saves decision to MemoryService. Logs all actions to MonitorService. Context injected into floating chatbot.
Total runtime: ~4 seconds
Rule-based logic (RMD calc, asset selection, tax analysis, trade simulation): < 0.1s
Ollama reinvestment advice (async, phi3 on CPU): ~4s with 4s timeout
Fallback if Ollama exceeds timeout: instant smart rule-based advice

14. Local Deployment Guide

Step 1 — Install Java 17

winget install Microsoft.OpenJDK.17 --accept-package-agreements --accept-source-agreements

Step 2 — Install Ollama + Pull Phi-3 Model

winget install Ollama.Ollama --accept-package-agreements --accept-source-agreements ollama pull phi3 # 3.8B model — runs on CPU, no GPU required ollama list # verify

Ollama starts automatically as a Windows background service on port 11434.

Step 3 — Install Maven & Node.js

# Maven Invoke-WebRequest -Uri "https://archive.apache.org/dist/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.zip" -OutFile "$env:USERPROFILE\maven.zip" -UseBasicParsing Expand-Archive "$env:USERPROFILE\maven.zip" -DestinationPath "$env:USERPROFILE\maven" $env:PATH = "$env:USERPROFILE\maven\apache-maven-3.9.6\bin;$env:PATH" # Node.js winget install OpenJS.NodeJS --accept-package-agreements --accept-source-agreements

Step 4 — Clone & Build
